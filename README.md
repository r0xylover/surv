# PWA-система обследования зданий и сооружений

Офлайн-приложение для инженерного обследования зданий и сооружений.

## Возможности

- работа как PWA;
- офлайн после первой загрузки;
- локальное хранение в IndexedDB;
- ведение дерева: объект → корпус → этаж → конструкция → дефект;
- размещение конструкций на схеме;
- карточки конструкций и дефектов;
- фотофиксация с автопривязкой;
- экспорт JSON, CSV, XLSX, ZIP;
- импорт проекта из JSON;
- деплой на GitHub Pages без сервера.

## Публикация на GitHub Pages

1. Создай новый репозиторий на GitHub.
2. Скопируй в него все файлы проекта.
3. Запушь ветку `main`.
4. На GitHub открой `Settings` → `Pages`.
5. В качестве источника выбери `GitHub Actions`.
6. Workflow `.github/workflows/deploy.yml` выполнит сборку и публикацию автоматически.

## Установка как PWA

### Android
1. Открой опубликованное приложение в Chrome.
2. Нажми меню браузера.
3. Выбери `Установить приложение`.

### iPhone
1. Открой опубликованное приложение в Safari.
2. Нажми `Поделиться`.
3. Выбери `На экран Домой`.
4. Запускай приложение именно с иконки на экране Домой, а не из вкладки браузера.
5. Для стабильной офлайн-работы один раз открой нужный проект при наличии интернета, затем дождись полной загрузки интерфейса.

### Windows / macOS
1. Открой приложение в Chrome или Edge.
2. Нажми значок установки в адресной строке.
3. Подтверди установку.

## Резервное копирование

1. Открой приложение.
2. Нажми кнопку `JSON` или `ZIP`.
3. Сохрани файл в безопасное место.
4. Для восстановления используй кнопку `Импорт`.

## Импорт и экспорт

- `JSON` — полный перенос структуры проекта;
- `Ведомости` — XLSX и CSV;
- `ZIP` — архив проекта со структурами и фотографиями.

## Обновление приложения

1. Закоммить изменения в `main`.
2. GitHub Actions автоматически пересоберет сайт.
3. На устройстве открой приложение заново.
4. Service worker подтянет новую версию автоматически.
